## 🎯 시스템 개요

이 시스템은 \*\*[시스템 A: 자동 감지]\*\*와 \*\*[시스템 B: 텔레그램 주문]\*\*이 통합된 하이브리드 자동매매 솔루션입니다.

  * **시스템 A (감시자)**: KIS 앱 등 외부에서 매수된 건을 자동으로 감지하여 매도 주문을 실행합니다.
  * **시스템 B (주문자)**: 텔레그램을 통해 설정한 목표가에 도달하면 자동으로 매수를 실행합니다.
  * **통합 제어**: 두 시스템은 **일일 8회 매매 제한**을 공유하며 유기적으로 작동합니다.

## ✨ 주요 기능

### 1\. 이원화된 매수 시스템

  * **자동 감지 (System A)**: MTS(앱)에서 수동으로 매수한 내역을 즉시 감지하여 매도 로직을 태웁니다.
  * **텔레그램 주문 (System B)**: 채팅으로 "애플 175불에 사줘"라고 명령하면, 가격 도달 시 자동 매수하고 매도 로직으로 넘깁니다.

### 2\. 자동 이익 실현 (Profit Taking)

  * 어떤 방식으로 매수하든 체결 즉시 **+6.0%** 가격에 지정가 매도 주문을 등록합니다.
  * 매도 미체결 시 10초마다 상태를 확인합니다.

### 3\. 통합 리스크 관리

  * **일일 8회 제한**: 과도한 매매를 방지하기 위해 하루 최대 8회(매도 기준)까지만 거래합니다.
  * **주말 자동 슬립**: 금요일 장 마감 후부터 월요일 프리마켓 시작 전까지 시스템이 자동으로 휴식합니다.
  * **Rate Limit 보호**: API 호출을 초당 37회 이하로 제한하여 차단을 방지합니다.

## 📜 운영 정책

1.  **운영 시간**: 미국 동부 시간(ET) **05:00 \~ 12:00** (한국 시간 저녁 \~ 새벽) 동안만 API를 호출합니다.
2.  **슬립 모드**:
      * 장 종료(ET 12:00) 후에는 **모든 대기 중인 텔레그램 주문이 자동 취소**됩니다. (매일 리셋)
      * **금요일 장 마감 후 \~ 월요일 05:00**까지는 시스템이 완전히 멈춥니다(주말 슬립).
3.  **8회 제한**: 하루에 매도 체결이 8번 발생하면, 그날은 더 이상 신규 매수나 텔레그램 주문을 받지 않습니다.
4.  **중복 허용**: 텔레그램으로 주문을 걸어뒀더라도, 앱에서 수동으로 동일 종목을 매수하면 **별개의 건으로 인식**하여 각각 처리합니다.
